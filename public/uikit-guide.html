Week 5 — UI Kit Setup

How to guide Claude to build your product the right way.

You don't need to be a developer to build something great. You need to know what to ask for, in what order, and how to tell if it looks right. This guide teaches you exactly that.

01 / THE RIGHT ORDER

Order matters more than speed.

The #1 mistake is asking Claude to build a full page right away. That's like decorating a house before pouring the foundation. Instead, work in layers — each one supports the next.

5

Full Screens

Your dashboard, login, settings — assembled from the pieces below

4

Sections & Layouts

Headers, sidebars, forms — combining your building blocks

3

Building Blocks

Buttons, cards, inputs, menus — reusable pieces that look & feel consistent

2

Your Visual Language

Colors, fonts, spacing, shadows — rules that everything follows

1

Your Brand Identity

What mood does your product convey? What colors represent you?

💡

Why this order?

When you build from the bottom up, every new screen becomes assembly — just arranging pieces that already exist and already look great together. Skip a layer, and every new feature becomes a fight with inconsistent colors, mismatched fonts, and things that just feel "off."

02 / STEP 1

Start by telling Claude who you are.

Before Claude writes a single line of code, it needs to understand your brand. Think of it like briefing a designer — the clearer your brief, the better the result.

🎨

Pick Your Colors

Choose a primary brand color and 1-2 accent colors. Find inspiration on sites you admire. You don't need to know hex codes — describe it: "a deep navy blue" or "warm coral, like Airbnb's orange."

🔠

Choose Your Font

Pick a font that matches your brand's personality. Modern and clean? Try something like Inter. Elegant and editorial? Try a serif. Just tell Claude the vibe — it will find the right one.

🌙

Light Mode, Dark Mode, or Both?

Decide upfront. If you want both, tell Claude from the start so it sets things up correctly. Adding it later is much harder.

✨

What's the Mood?

Is your product serious and professional? Fun and playful? Minimal and calm? This guides every visual decision — from button shapes to spacing between elements.

Example What to tell Claude in Step 1
"I'm building a [type of product]. The brand uses [color description] as the main color. The mood should feel [professional / playful / minimal / bold]. I want to support [light mode / dark mode / both]. Please set up my project with a design system — define all the colors, fonts, and spacing as a visual language that every part of the app will follow. Make sure the colors work well together and are accessible."
⚠️

Why this is worth 10 minutes of thought

If you skip this and go straight to "build me a dashboard," Claude will pick random colors and fonts each time you ask for something new. Your app ends up looking like five different apps glued together.

03 / STEP 2

Ask Claude to build your pieces first.

Before building pages, ask Claude to create small, reusable pieces — buttons, cards, inputs. These are your building blocks. Every screen you build later will use them.

What to ask Claude for each block

🔄

Multiple Variations

A button isn't just one button. Ask for variations: a main action button, a secondary/outline button, and a subtle/ghost button. Same for sizes: small, medium, large.

🎨

Consistent Styling

Tell Claude: "Use the colors and fonts from the design system we already set up." This ensures every piece looks like it belongs to the same product.

📦

Keep Them Independent

Each piece should work on its own. A Button shouldn't depend on a specific page or feature. This way you can use it anywhere.

✅

Test Each One

Before moving on, look at each piece. Does the button look clickable? Does it have a hover effect? Does the card look clean? Fix the small stuff now — it saves time later.

Example What to tell Claude in Step 2
"Now create a set of reusable building blocks using the design system we defined. Start with: a Button (with filled, outlined, and subtle styles in small/medium/large sizes), a Card (a container with a title, content area, and subtle shadow), and a Text Input (with a label, placeholder, and error state). Make sure they all use the same colors and fonts from our design system, and they should work well in [light / dark / both] mode."
📐

Start small — 3 blocks is enough

For this week, you only need a Button, a Card, and an Input. Don't try to build 15 pieces upfront. Add more as your product actually needs them.

04 / STEP 3

Now assemble your screens.

With your identity defined and your building blocks ready, now you tell Claude to put them together into real screens. This is where things come alive.

A

Describe the screen, not the code

Tell Claude what the user should see: "A clean dashboard with a header at the top, a navigation sidebar, and a main area that shows cards with key metrics." You don't need to say how to build it — just what it should look and feel like.

B

Remind Claude to use your existing pieces

Say: "Use the Button, Card, and Input components we already built." This is the magic — instead of generating new random elements, Claude reuses the building blocks that already match your brand.

C

Add the AI interaction

Once your screen looks right, add the smart part: "Add a text input where the user can type a question, a send button, and a card below that shows the AI response." Start with one simple interaction that works end-to-end.

D

Refine by describing what feels off

Don't restart the conversation. Instead say things like: "The spacing between the cards feels too tight," or "The header doesn't stand out enough," or "Make the button more prominent." Claude understands visual feedback.

Example What to tell Claude in Step 3
"Build me the main screen of my app. It should have a header with my product name and a navigation bar, and the main area should show [describe what the user sees]. Use the components we already created — the cards, buttons, and inputs. Make sure it feels [clean / spacious / professional] and follows our design system."
05 / FIND REFERENCES

Don't design from scratch. Find what works and bring it.

The best products aren't invented — they're inspired. Before asking Claude to build something, find examples of apps that do it well and share them as references.

🔍

Browse Real Products

Before building, spend 10 minutes looking at products you admire. How does Notion handle navigation? How does Linear display a dashboard? How does Stripe handle forms? Screenshot what you like.

🎨

Explore Design Galleries

Sites like Dribbble, Mobbin, or Godly showcase the best UI designs. Search for your type of product — "analytics dashboard," "AI chat interface," "settings page" — and save what resonates.

💬

Tell Claude What You Found

Share your references: "I like how Notion uses a clean sidebar with icons," or "I want cards similar to Linear's dashboard — minimal with subtle shadows." The more specific, the better the output.

🧠

Describe the UX, Not Just the Look

Great design is about how things feel, not just how they look. "When I hover, buttons should have a smooth transition," or "The page should feel spacious, not crowded." Claude can deliver on UX cues.

Example Bringing references to Claude
"For the navigation, I want something similar to Notion's sidebar — clean, with icons next to each item, and it should collapse on mobile. For the main content area, I like how Linear shows cards in a grid — minimal design, lots of white space, subtle shadows on hover. Please follow these references while using our existing design system."
🚀

The Reference Workflow

Find 2-3 products you admire → Screenshot the parts you like → Describe what you like about them to Claude → Let Claude build your version. This is how professionals work — they research before they create.

06 / QUALITY CHECK

How to tell if it's built well.

You don't need to read code to check quality. Here's what to look for visually — if any of these feel off, tell Claude what to fix.

🎨 Color Consistency

  • Same blue used everywhere for primary actions?
  • Text colors feel consistent across all screens?
  • No random colors that don't belong to your palette?
  • Dark mode looks intentional, not just inverted?

🔠 Typography

  • Only 1-2 fonts used across the whole app?
  • Headlines are clearly bigger than body text?
  • Text is easy to read — not too small, not too light?
  • Same font styling for similar elements?

📐 Spacing & Alignment

  • Elements feel evenly spaced — nothing cramped?
  • Cards, buttons, and inputs are aligned properly?
  • Consistent padding inside cards and sections?
  • The page has "room to breathe" — not cluttered?

🖱️ Interactions

  • Buttons change on hover — visual feedback?
  • Clickable things look clickable?
  • Transitions feel smooth, not jumpy?
  • Focus states visible when tabbing with keyboard?

🧩 Consistency

  • All buttons look the same style across pages?
  • Cards have the same shadow and border-radius?
  • Navigation looks identical on every screen?
  • If a pattern is used once, it's used everywhere?

📱 Usability

  • Layout makes sense — user knows where to look?
  • Most important content is the most visible?
  • It feels like one product, not parts glued together?
  • A new user could figure out what to do?
⚠️

When something feels off, describe the feeling

You don't need technical words. Tell Claude: "The buttons feel too small," "There's too much empty space on the right," "The cards don't feel connected to each other," "The header looks disconnected from the rest." Claude understands this language perfectly.

07 / THIS WEEK

Your Week 5 game plan.

Follow this order. Broken parts are fine — the goal is a working skeleton that proves the foundation is right.

1
Set up your project and CLAUDE.md Ask Claude to scaffold your project and create a CLAUDE.md file that describes your stack, structure, and rules. This file makes Claude smarter about your project in every future conversation.
2
Define your brand identity and visual language Tell Claude your colors, fonts, and mood. Ask it to set up a design system with your visual language so every element follows the same rules.
3
Build 2-3 building blocks Ask Claude to create a Button, a Card, and an Input — with variations. Check that they look consistent and match your brand.
4
Assemble your first screen Tell Claude to build a layout using the pieces you created. A header, a main area, and your content. It should feel like a real product.
5
Wire up one AI interaction Add a simple flow: user types something → it goes to the AI → the response shows on screen. It can be simple — it just has to work.
6
Do a quality check Use the audit checklist above. Are colors consistent? Fonts matching? Spacing even? Fix what feels off before moving on.
7
Record a walkthrough Screen recording or live link showing what you built. Broken parts are expected. Progress is the point.
🔥

Remember: Foundation → Blocks → Screens → AI

Don't skip ahead. The whole point of this order is that each step makes the next one easier. Trust the process.
